# 鉴权限流管理后台界面操作说明

本文面向管理员、运营人员和服务负责人，说明如何通过内置管理后台完成用户、APP、OIDC Client、服务注册、限流、黑白名单、日志和统计管理。

管理后台地址：

```text
https://auth-limit.baichengedu.com/ui/
```

文档中的账号、Token、Secret、服务 ID 均为占位符。请勿在截图、群聊或文档中传播生产可用 Secret。

## 1. 登录后台

1. 打开 `https://auth-limit.baichengedu.com/ui/`。
2. 输入管理员或已授权用户的用户名和密码。
3. 点击登录。
4. 登录成功后进入工作台，顶部显示“已登录”。

会话说明：

| 场景 | 处理方式 |
| --- | --- |
| 页面刷新 | 浏览器会从本地存储恢复会话 |
| Access Token 接近过期 | 后台会自动刷新并保存新的 Access Token |
| Refresh Token 失效 | 操作接口会返回登录错误，需要重新登录 |
| 退出登录 | 点击右上角退出，系统会吊销当前 Token |

如果连续输错密码导致账号锁定，请等待锁定时间结束或联系管理员处理。

## 2. 用户管理

进入“用户管理”菜单，可查看和维护系统用户。

### 2.1 注册用户

1. 点击“注册用户”。
2. 填写用户名、密码、显示名。
3. 点击提交。

填写规则：

| 字段 | 规则 |
| --- | --- |
| 用户名 | 3-20 位字母、数字或下划线 |
| 密码 | 8-20 位，包含大写字母、小写字母、数字和特殊字符 |
| 显示名 | 可选；不填时默认使用用户名 |

新用户默认启用，但没有管理权限，需要通过角色分配获得权限。

### 2.2 编辑用户

1. 在用户列表找到目标用户。
2. 点击“编辑”。
3. 修改显示名。
4. 点击提交。

### 2.3 启用或禁用用户

1. 在目标用户行点击“禁用”或“启用”。
2. 操作成功后列表会刷新。

禁用用户后，该用户不能继续登录。管理员不能禁用自己。

### 2.4 修改密码

1. 在目标用户行点击“密码”。
2. 输入新密码。
3. 点击提交。

管理员可以重置其他用户密码。普通用户修改自己的密码时，API 层需要旧密码；当前轻量后台主要面向管理员操作。

### 2.5 分配角色

1. 在目标用户行点击“角色”。
2. 在弹窗中选择一个或多个角色。
3. 点击提交。

角色变更后，用户需要重新登录或刷新 Token，新的权限才会生效。

### 2.6 删除用户

1. 在目标用户行点击“删除”。
2. 确认删除。

管理员不能删除自己。删除用户前，请确认该用户不再负责任何服务、APP 或 OIDC Client。

## 3. 角色权限

进入“角色权限”菜单，可维护角色和权限点。

### 3.1 查看权限点

创建或编辑角色时，页面会自动加载系统权限点。当前权限包括：

| 权限点 | 用途 |
| --- | --- |
| `user:manage` | 用户管理 |
| `app:manage` | APP 管理 |
| `role:manage` | 角色权限管理 |
| `oidc:manage` | OIDC Client 管理 |
| `service:manage` | 服务管理 |
| `limit:manage` | 限流规则管理 |
| `blacklist:manage` | 黑白名单管理 |
| `log:read` | 日志查询 |
| `health:read` | 健康检测查询 |
| `statistics:read` | 限流统计查询 |

### 3.2 创建角色

1. 点击“创建角色”。
2. 填写角色标识、名称、描述。
3. 勾选权限。
4. 点击提交。

角色标识是系统内部唯一编码，例如 `service_operator`。创建后应避免频繁改名，便于审计和排查。

### 3.3 编辑角色

1. 在角色列表中点击“编辑”。
2. 修改名称、描述或权限。
3. 点击提交。

系统内置角色会标记为“系统=是”，不可编辑或删除。内置超级管理员角色拥有全部权限。

### 3.4 删除角色

1. 在非系统角色行点击“删除”。
2. 确认删除。

删除角色会影响已分配该角色的用户或 APP。删除前请确认没有生产接入依赖该角色。

## 4. APP 管理

进入“APP 管理”菜单，可创建机器调用身份，用于 M2M 签名鉴权。

### 4.1 创建 APP

1. 点击“创建 APP”。
2. 填写名称，例如 `order-service`。
3. 点击提交。
4. 系统弹出一次性 `AppSecret`。
5. 立即保存 Secret 到安全的密钥管理系统。

注意：

| 事项 | 说明 |
| --- | --- |
| `AppSecret` 只显示一次 | 关闭弹窗后无法再次查看 |
| 泄露后要重置 | 重置后旧 Secret 立即失效 |
| APPID 用于请求头 | M2M 请求头使用 `appId`，值为列表里的 APPID |

### 4.2 编辑 APP

1. 点击目标 APP 的“编辑”。
2. 修改名称或状态。
3. 点击提交。

状态设置为 `disabled` 后，该 APP 的 M2M 鉴权会失败。

### 4.3 重置密钥

1. 点击目标 APP 的“重置密钥”。
2. 系统弹出新的 `AppSecret`。
3. 立即通知调用方更新配置。

重置密钥会使旧密钥失效。建议先协调调用方更新时间窗口，避免业务中断。

### 4.4 分配角色

1. 点击目标 APP 的“角色”。
2. 选择一个或多个角色。
3. 点击提交。

当前 M2M 鉴权负责确认 APP 身份和签名有效。业务侧如需按 APP 角色做细粒度授权，应结合 APPID 和角色配置扩展策略。

### 4.5 删除 APP

1. 点击目标 APP 的“删除”。
2. 确认删除。

删除后，该 APP 无法继续通过 M2M 鉴权。

## 5. OIDC Client

进入“OIDC Client”菜单，可为 APISIX 或业务系统创建 OAuth2/OIDC 客户端。

### 5.1 创建 OIDC Client

1. 点击“创建 OIDC Client”。
2. 填写名称。
3. 填写回调地址 `redirectUri`。
4. 点击提交。
5. 保存弹窗里的一次性 `ClientSecret`。

回调地址必须与业务系统实际使用的地址完全一致，包括协议、域名、路径和端口。

### 5.2 编辑 OIDC Client

1. 点击目标 Client 的“编辑”。
2. 修改名称、回调地址或状态。
3. 点击提交。

状态设置为 `disabled` 后，该 Client 不能继续换取 Token。

### 5.3 重置 Client Secret

1. 点击“重置密钥”。
2. 保存新的 `ClientSecret`。
3. 同步更新 APISIX 或业务系统配置。

重置后旧 Secret 立即失效。

### 5.4 对接业务系统

把以下信息交给业务系统或 APISIX 配置负责人：

| 配置项 | 值 |
| --- | --- |
| Issuer | `https://auth-limit.baichengedu.com` |
| Discovery | `https://auth-limit.baichengedu.com/.well-known/openid-configuration` |
| Authorization Endpoint | `https://auth-limit.baichengedu.com/oauth2/authorize` |
| Token Endpoint | `https://auth-limit.baichengedu.com/oauth2/token` |
| JWKS URI | `https://auth-limit.baichengedu.com/oauth2/jwks` |
| UserInfo Endpoint | `https://auth-limit.baichengedu.com/oauth2/userinfo` |
| Client ID | 页面返回的 `clientId` |
| Client Secret | 创建或重置时保存的一次性 Secret |

## 6. 服务管理

进入“服务管理”菜单，可注册下游服务、审核服务并查看健康状态。

### 6.1 注册服务

1. 点击“注册服务”。
2. 填写名称。
3. 填写编码。
4. 填写服务地址 `baseUrl`。
5. 填写健康路径，默认 `/health`。
6. 填写检测间隔秒数。
7. 点击提交。

字段说明：

| 字段 | 说明 |
| --- | --- |
| 名称 | 给人看的服务名称 |
| 编码 | 全局唯一服务编码，例如 `order-service` |
| 服务地址 | 下游服务基础地址，例如 `https://orders.example.com` |
| 健康路径 | 健康检测路径，例如 `/health` |
| 检测间隔秒 | 当前配置允许 10-300 秒 |

普通用户注册的服务状态为“待审核”。管理员注册服务会自动审核。

### 6.2 审核服务

1. 管理员进入“服务管理”。
2. 找到待审核服务。
3. 点击“审核”。
4. 状态变为 `approved`。

审核通过不代表立即进入服务发现。服务还需要健康检测通过，`healthStatus` 变为 `healthy`。

### 6.3 编辑服务

1. 点击目标服务的“编辑”。
2. 修改名称、服务地址、健康路径或检测间隔。
3. 管理员还可以修改服务状态和健康状态。
4. 点击提交。

如果服务地址变化，应同时确认 APISIX upstream 或同步脚本是否已更新。

### 6.4 按健康状态筛选

页面支持按健康状态筛选：

| 状态 | 说明 |
| --- | --- |
| `healthy` | 健康检测通过 |
| `degraded` | 亚健康 |
| `unhealthy` | 异常 |
| `unknown` | 尚未检测或状态未知 |

服务发现只返回已审核、状态为 `approved`、健康状态为 `healthy` 的服务。

### 6.5 删除服务

1. 点击目标服务的“删除”。
2. 确认删除。

删除服务前，请先确认 APISIX 路由、限流规则、黑白名单和统计查询不再依赖该服务。

## 7. 限流规则

进入“限流规则”菜单，可配置服务级动态限流。

### 7.1 创建限流规则

1. 点击“创建限流规则”。
2. 选择服务。
3. 选择维度。
4. 选择粒度。
5. 填写容量和每秒令牌数。
6. 按需填写拉黑触发次数、封禁秒数。
7. 勾选启用。
8. 点击提交。

字段说明：

| 字段 | 说明 |
| --- | --- |
| 服务 | 限流规则归属服务 |
| 维度 | `ip`、`user_id`、`app_id`、`path` |
| 粒度 | `second`、`minute`、`hour`、`day` |
| 容量 | 令牌桶容量，代表可承受突发请求数 |
| 每秒令牌 | 每秒补充令牌数 |
| 拉黑触发次数 | 被限流达到次数后自动临时拉黑，`0` 表示不触发 |
| 封禁秒数 | 临时拉黑时长，`0` 表示不触发 |
| 启用 | 是否立即生效 |

同一服务、同一维度、同一粒度只能有一条启用规则。重复启用会提示冲突。

### 7.2 编辑限流规则

1. 点击目标规则的“编辑”。
2. 修改维度、粒度、容量、速率或启用状态。
3. 点击提交。

修改启用规则会影响线上流量，请先确认业务流量峰值。

### 7.3 删除限流规则

1. 点击目标规则的“删除”。
2. 确认删除。

删除服务级规则后，如果仍有请求进入限流校验，系统会使用全局默认限流配置。

## 8. 黑白名单

进入“黑白名单”菜单，可维护拦截和免限流策略。

### 8.1 新增黑名单

1. 点击“新增名单”。
2. 名单类型选择“黑名单”。
3. 选择服务。
4. 选择主体类型。
5. 填写主体值。
6. 填写原因。
7. 选择是否永久黑名单。
8. 如非永久，填写过期时间。
9. 点击提交。

黑名单命中后，限流校验返回 403。

### 8.2 新增白名单

1. 点击“新增名单”。
2. 名单类型选择“白名单”。
3. 选择服务。
4. 选择主体类型。
5. 填写主体值。
6. 填写原因和过期时间。
7. 点击提交。

白名单命中后跳过限流。白名单优先级高于黑名单。

### 8.3 主体类型

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| `ip` | 客户端 IP | `203.0.113.10` |
| `user` | 用户 ID | `<USER_ID>` |
| `app` | APP ID | `<APP_ID>` |
| `token` | Token 标识 | `<TOKEN_ID>` |

当前白名单支持 `ip/user/app`，黑名单支持 `ip/user/app/token`。

### 8.4 删除名单

1. 在名单列表找到目标记录。
2. 点击“删除”。
3. 确认删除。

新增和删除黑白名单当前要求管理员身份。普通授权用户可查看其权限允许的页面，但创建名单时可能因非管理员身份失败。

## 9. 日志查询

进入“日志查询”菜单，可查看操作、鉴权、限流和健康检测日志。

### 9.1 筛选日志

1. 选择日志类型。
2. 按需选择服务。
3. 按需选择结果：成功或失败。
4. 点击“筛选”。

日志类型：

| 类型 | 说明 |
| --- | --- |
| 操作 | 后台和管理 API 的增删改操作 |
| 鉴权 | 登录、M2M 等认证事件 |
| 限流 | 限流校验记录 |
| 健康 | 服务健康检测记录 |

### 9.2 排查建议

| 现象 | 查看日志 |
| --- | --- |
| 用户无法登录 | 鉴权日志，筛选失败 |
| 管理操作失败 | 操作日志，查看资源和动作 |
| 请求被限流 | 限流日志，按服务和维度查看 |
| 服务未进入发现列表 | 健康日志，确认最近一次检测状态 |

## 10. 限流统计

进入“限流统计”菜单，可按服务和维度查看限流汇总。

### 10.1 查询统计

1. 选择服务。
2. 选择维度，可选全部、IP、用户、APP、接口。
3. 点击“筛选”。

列表字段：

| 字段 | 说明 |
| --- | --- |
| 时间 | 统计桶时间 |
| 服务 | 服务名称和编码 |
| 维度 | 限流维度 |
| 请求数 | 总请求量 |
| 拦截数 | 被限流或拉黑拦截的请求数 |

如果刚发生请求但统计暂未出现，可等待短时间后刷新。

## 11. 健康检测

进入“健康检测”菜单，可查看下游服务健康检测记录。

### 11.1 查看健康记录

1. 选择服务。
2. 点击“筛选”。
3. 查看状态、HTTP 状态码、耗时和错误信息。

状态说明：

| 状态 | 说明 |
| --- | --- |
| `healthy` | 健康检测成功 |
| `unhealthy` | 健康检测失败 |
| `unknown` | 尚无有效检测结果 |

服务发现依赖健康状态。若服务已审核但仍不能被发现，优先查看此页面。

## 12. APISIX 接入向导

通过界面完成 APISIX 接入前置准备，再交给开发者按 API 操作版配置 APISIX。

### 12.1 OIDC 鉴权接入准备

1. 进入“OIDC Client”。
2. 创建 Client。
3. 填写 APISIX 或业务系统回调地址。
4. 保存 `clientId` 和一次性 `clientSecret`。
5. 把以下信息交给 APISIX 配置负责人：

```text
Discovery: https://auth-limit.baichengedu.com/.well-known/openid-configuration
Client ID: <CLIENT_ID>
Client Secret: <CLIENT_SECRET>
Redirect URI: <REDIRECT_URI>
```

### 12.2 服务注册和发现准备

1. 进入“服务管理”。
2. 注册业务服务。
3. 管理员审核服务。
4. 等待健康检测通过。
5. 确认健康状态为 `healthy`。
6. 把服务 ID、服务编码、服务地址交给 APISIX 同步脚本维护者。

服务发现接口只返回健康且已审核服务。APISIX 同步脚本应使用 API 操作版中的 `/api/services/discover`。

### 12.3 限流接入准备

1. 进入“限流规则”。
2. 为目标服务创建规则。
3. 确认维度与 APISIX 会传入的字段一致：
   - IP 限流需要传 `ip`。
   - 用户限流需要鉴权后能得到 `userId`。
   - APP 限流需要传 `appId`。
   - 接口限流需要传 `path`。
4. 启用规则。
5. 在“限流统计”观察请求数和拦截数。

### 12.4 M2M 接入准备

1. 进入“APP 管理”。
2. 创建 APP。
3. 保存 `appId` 和一次性 `appSecret`。
4. 把凭据交给调用方，由调用方按 API 操作版的 M2M 签名算法生成请求头。

### 12.5 黑白名单策略准备

1. 进入“黑白名单”。
2. 选择服务。
3. 添加需要拦截的 IP、用户、APP 或 Token。
4. 添加需要跳过限流的 IP、用户或 APP。
5. 用 API 操作版中的 `/oidc/limit/verify` 验证命中效果。

## 13. 操作安全建议

| 建议 | 说明 |
| --- | --- |
| 及时保存一次性 Secret | APP Secret 和 Client Secret 只在创建或重置时显示一次 |
| 最小权限授权 | 普通服务负责人只分配所需权限，不直接授予管理员角色 |
| 变更前确认依赖 | 删除服务、APP、角色前确认 APISIX 和业务系统不再依赖 |
| 高峰期谨慎改限流 | 限流规则实时生效，建议先在低峰期调整 |
| 使用日志闭环排查 | 登录失败看鉴权日志，限流问题看限流日志和统计，服务发现问题看健康检测 |
